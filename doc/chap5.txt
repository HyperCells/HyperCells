  [1X5.1 [33X[0;0YSupercell Model Graphs[133X[101X
  
  [33X[0;0YSupercell  model  graphs  are used to describe the extension of a cell model
  graph  defined on a primitive cell to a supercell. The vertices and edges of
  the supercell model graph retain information about the vertices and edges of
  the  primitive  cell  model  graph  and  the  elements of the quotient group
  [23X\Gamma_\mathrm{pc}/\Gamma_\mathrm{sc}[123X distinguishing the different copies of
  the primitive cell in the supercell.[133X
  
  [33X[0;0YSupercell   model   graphs   are   implemented   as   objects   of  category
  [10XTGSuperCellModelGraph[110X with the following components:[133X
  
  [30X    [33X[0;6Y[2XGetTGPrimitiveCell[102X ([14X5.3-5[114X): the associated [10XTGCell[110X object (see [14X3.1[114X) for
        the primitive cell[133X
  
  [30X    [33X[0;6Y[2XGetTGSuperCell[102X ([14X5.3-6[114X): the associated [10XTGCell[110X object (see [14X3.1[114X) for the
        supercell[133X
  
  [30X    [33X[0;6Y[2XCellCenter[102X ([14X5.3-4[114X): the cell center of the supercell model graph[133X
  
  [30X    [33X[0;6Y[2XCellEmbedding[102X  ([14X5.3-14[114X):  the  embedding  of the primitive cell in the
        supercell as a [10XTGCellEmbedding[110X object (see [14X5.4[114X)[133X
  
  [30X    [33X[0;6Y[2XModelType[102X   ([14X5.3-8[114X):   identifier   of   the   type  of  model,  e.g.,
        tessellation, kagome etc.[133X
  
  [30X    [33X[0;6Y[2XCellVertices[102X ([14X5.3-9[114X): the vertices of the supercell model graph, i.e.,
        the sites of the tight-binding model[133X
  
  [30X    [33X[0;6Y[2XCellVertexPositions[102X  ([14X5.3-10[114X):  the  positions  of the vertices of the
        supercell model graph as elements of the proper triangle group[133X
  
  [30X    [33X[0;6Y[2XCellEdges[102X  ([14X5.3-11[114X): the edges of the supercell model graph, i.e., the
        hoppings  of the tight-binding model, including potential translations
        associated with them[133X
  
  [30X    [33X[0;6Y[2XCellFaces[102X  ([14X5.3-13[114X): the faces of the supercell model graph, i.e., the
        plaquettes of the tight-binding model[133X
  
  [33X[0;0Yand is printed in the form[133X
  
  [4X[32X  Example  [32X[104X
    [4X[28XTGSuperCellModelGraph([128X[104X
    [4X[28X  primitive cell = TGCell( ... ),[128X[104X
    [4X[28X  supercell = TGCell( ... ),[128X[104X
    [4X[28X  center = w,[128X[104X
    [4X[28X  type = <type>,[128X[104X
    [4X[28X  vertices = [ [ w1, gi1 ], [ w2, gi2 ], ... ],[128X[104X
    [4X[28X  edges = [ [ v1, v2, s, gam ], ... ],[128X[104X
    [4X[28X  faces = [ [ [ e1, o1 ], [ e2, o2 ], ... ], ... ][128X[104X
    [4X[28X)[128X[104X
  [4X[32X[104X
  
  [33X[0;0YSee the corresponding references above for details.[133X
  
  
  [1X5.2 [33X[0;0YConstructing Supercell Models[133X[101X
  
  [33X[0;0YThere  are  two functions for constructing supercell models based on a given
  model graph:[133X
  
  [30X    [33X[0;6Y[2XTGSuperCellModelGraph[102X ([14X5.2-1[114X): constructs a supercell model graph from
        a  model  graph and a supercell choosing representatives such that the
        resulting supercell is a symmetric connected cell.[133X
  
  [30X    [33X[0;6Y[2XRandomTGSuperCellModelGraph[102X  ([14X5.2-2[114X):  constructs  a  supercell  model
        graph  from  a model graph and relators specifying the supercell. Note
        that  the  resulting  supercell  is neither symmetric nor connected in
        general.  Usually,  the  representatives  are automatically chosen and
        arbitrary, but they can be explicitly passed if desired.[133X
  
  [1X5.2-1 TGSuperCellModelGraph[101X
  
  [33X[1;0Y[29X[2XTGSuperCellModelGraph[102X( [3Xmodel[103X, [3Xsc[103X ) [32X function[133X
  [6XReturns:[106X  [33X[0;10Ymodel graph as [10XTGSuperCellModelGraph[110X object (see [14X5.1[114X).[133X
  
  [33X[0;0YConstructs the supercell model graph obtained from extending the model [3Xmodel[103X
  given  as a [10XTGCellModelGraph[110X object (see [14X4.1[114X) to the supercell [3Xsc[103X given as a
  [10XTGCell[110Xobject  (see  [14X3.1[114X).  The  resulting  supercell  is  both symmetric and
  connected.[133X
  
  [1X5.2-2 RandomTGSuperCellModelGraph[101X
  
  [33X[1;0Y[29X[2XRandomTGSuperCellModelGraph[102X( [3Xmodel[103X, [3Xscquotient[103X[, [3XGAMgens[103X[, [3XTDGAM[103X[, [3XTGAMs[103X]]] ) [32X function[133X
  [6XReturns:[106X  [33X[0;10Ycell graph as [10XSection_TGSuperCellModelGraph[110X object (see [14X5.1[114X).[133X
  
  [33X[0;0YConstructs the supercell model graph obtained from extending the model [3Xmodel[103X
  given  as  a [10XTGCellModelGraph[110X object (see [14X4.1[114X) to the supercell specified by
  the  [10XTGQuotient[110X  [3Xscquotient[103X (see [14X2.3[114X). Generally, the resulting supercell is
  neither symmetric nor connected.[133X
  
  [33X[0;0YIf  desired,  explicit,  e.g.,  symmetric,  choices  for  the  generators of
  supercell           translation           group          [23X\Gamma_\mathrm{sc}[123X,
  [23XT_{\Delta^+}(\Gamma_\mathrm{sc})[123X,                                        and
  [23XT_{\Gamma_\mathrm{pc}}(\Gamma_\mathrm{sc})[123X   can   be   given   as  optional
  arguments [3XGAMgens[103X, [3XTDGAM[103X, and [3XTGAMs[103X, respectively.[133X

  [1X5.3-11 CellEdges[101X
  
  [33X[1;0Y[29X[2XCellEdges[102X( [3Xmodel[103X ) [32X operation[133X
  
  [33X[0;0Yreturns the edges of the model graph as a list.[133X
  
  [33X[0;0YEach  edge  is of the form [10X[ v1, v2, tag, gam ][110X with [10Xv1[110X, [10Xv2[110X the positions of
  the  vertices  in  [10XCellVertices([110X[3Xmodel[103X[10X)[110X  connected  by the edge, [10Xtag[110X uniquely
  identifying  the  edge,  and  [10Xgam[110X  the  translation  relating  the unit cell
  containing  [10Xv1[110X  to  the  unit cell containing [10Xv2[110X, as element of the finitely
  presented  group  [10XFpGroup(TGCellTranslationGroup(GetTGCell([110X[3Xmodel[103X[10X)))[110X.  If  [10Xv1[110X
  corresponds  to  the  vertex  [23X(w,  [g_{u_w}]_w)[123X  and  [10Xv2[110X  to the vertex [23X(w',
  [g_{\tilde{u}_{w'}}]_{w'})[123X,  then  [10Xgam[110X corresponds to the translation [23X\gamma[123X
  such that[133X
  
  
  [24X[33X[0;6Yg_{\tilde{u}_{w'}} = g_{u_w}\gamma.[133X
  
  [124X
  
  [33X[0;0YThe  tag  [10Xtag[110X  is  of  the  form  [10X[  v1pc, v2pc, tagpc ][110X with [10Xv1pc[110X, [10Xv2pc[110X the
  positions  of  the  vertices in the primitive cell, and [10Xtagpc[110X the tag of the
  edge  in  the  primitive  cell  (see  [14X4.2[114X).  Thus,  [10Xtag[110X specifies (up to the
  translation) the edge in the primitive cell.[133X
  
  [1X5.3-12 CellEdgeTranslations[101X
